variables and data types

Creating variables and data types is the first step in learning C#.

Variables are used to store data and can be of different types.

The most common data types are int, float, string and bool.

int is used to store whole numbers, float is used to store decimal numbers, string is used to store text and bool is used to store true or false values.

You can create a variable by using the following syntax:

int myInt = 5;

This creates a variable called myInt of type int and assigns it the value 5.

You can also create multiple variables of the same type in one line by using commas:

int myInt1 = 5, myInt2 = 10, myInt3 = 15;

This creates three variables of type int and assigns them the values 5, 10 and 15.

You can also create variables of different types in one line by using semicolons:

int myInt = 5; float myFloat = 5.5f; string myString = "Hello"; bool myBool = true;

This creates four variables of different types and assigns them the values 5, 5.5, "Hello" and true.

You can also create variables without assigning them a value by using the following syntax:
int myInt;

This creates a variable called myInt of type int without assigning it a value.

You can assign a value to the variable later by using the following syntax:

myInt = 5;

This assigns the value 5 to the variable myInt.

You can also change the value of a variable by using the following syntax:

myInt = 10;

This changes the value of the variable myInt to 10.

You can also use variables in calculations by using the following syntax:

int myInt1 = 5; int myInt2 = 10; int myInt3 = myInt1 + myInt2;

This creates two variables of type int and assigns them the values 5 and 10.

It then creates a third variable of type int and assigns it the value of the first two variables added together.

You can also use variables in calculations with different data types by using the following syntax:

int myInt1 = 5; float myFloat1 = 5.5f; float myFloat2 = myInt1 + myFloat1;

This creates two variables of type int and float and assigns them the values 5 and 5.5.

It then creates a third variable of type float and assigns it the value of the first two variables added together.

You can also use variables in calculations with different data types by using the following syntax:

int myInt1 = 5; string myString1 = "Hello"; string myString2 = myString1 + " World";

This creates two variables of type int and string and assigns them the values 5 and "Hello".

It then creates a third variable of type string and assigns it the value of the first two variables added together.

You can also use variables in calculations with different data types by using the following syntax:

int myInt1 = 5; bool myBool1 = true; bool myBool2 = myBool1 && (myInt1 > 0);

This creates two variables of type int and bool and assigns them the values 5 and true.

It then creates a third variable of type bool and assigns it the value of the first two variables added together.

You can also use variables in calculations with different data types by using the following syntax:

int myInt1 = 5; bool myBool1 = true; bool myBool2 = myBool1 || (myInt1 > 0);

This creates two variables of type int and bool and assigns them the values 5 and true.

Control structures: conditionals, loops

Control structures are used to control the flow of a program.

They are used to make decisions and repeat actions.

The most common control structures are conditionals and loops.

Conditionals are used to make decisions based on a condition.

They are used to execute a block of code if a condition is true and skip it if the condition is false.

The most common conditional is the if statement.

The if statement is used to execute a block of code if a condition is true.

The syntax for the if statement is as follows:

if (condition) { // code to execute if condition is true }

This executes the block of code inside the curly braces if the condition is true.

You can also use the else statement to execute a block of code if the condition is false.

The syntax for the else statement is as follows:

if (condition) { // code to execute if condition is true } else { // code to execute if condition is false }

This executes the block of code inside the first curly braces if the condition is true and the block of code inside the second curly braces if the condition is false.

You can also use the else if statement to check multiple conditions.

The syntax for the else if statement is as follows:

if (condition1) { // code to execute if condition1 is true } else if (condition2) { // code to execute if condition2 is true } else { // code to execute if both conditions are false }

This executes the block of code inside the first curly braces if condition1 is true, the block of code inside the second curly braces if condition2 is true and the block of code inside the third curly braces if both conditions are false.

You can also use the switch statement to check multiple conditions.

The syntax for the switch statement is as follows:

switch (variable) { case value1: // code to execute if variable is equal to value1 break;
case value2: // code to execute if variable is equal to value2 break; default: // code to execute if variable is not equal to any of the values }

This executes the block of code inside the curly braces if the variable is equal to any of the values.

You can also use the break statement to exit the switch statement.

The break statement is used to exit a loop or switch statement.

The syntax for the break statement is as follows:

break;

This exits the loop or switch statement.

You can also use the continue statement to skip the current iteration of a loop.

The syntax for the continue statement is as follows:

continue;

This skips the current iteration of the loop and continues to the next iteration.

You can also use the return statement to exit a method and return a value.

The syntax for the return statement is as follows:

return value;

This exits the method and returns the value.

You can also use the return statement to exit a method without returning a value.

The syntax for the return statement is as follows:

return;

This exits the method without returning a value.

You can also use the break statement to exit a loop or switch statement.

The syntax for the break statement is as follows:

break;

This exits the loop or switch statement.

The project also includes a simple UI that displays the current position of the character sprite.

You can download the project from the Asset Store or create it from scratch using the instructions below.

To create the project from scratch, follow these steps:

  1. Create a new Unity project.
  2. Create a new scene.
  3. Add a new GameObject to the scene.
  4. Add a new Sprite Renderer component to the GameObject.
  5. Add a new Rigidbody2D component to the GameObject.
  6. Add a new Box Collider2D component to the GameObject.
  7. Add a new script to the GameObject.
  8. Open the script in your code editor.
  9. Copy and paste the code below into the script.
  10. Save the script and return to Unity.
  11. Press play to test the project.


The code for the project is as follows:

using UnityEngine;

using System.Collections;

public class PlayerController : MonoBehaviour

{

public float speed = 5.0f;

private Rigidbody2D rb;

private Vector2 moveInput;

private Vector2 moveVelocity;



void Start()

{

rb = GetComponent();

}


void Update()

{

moveInput = new Vector2(Input.GetAxis("Horizontal"), Input.GetAxis("Vertical"));

moveVelocity = moveInput.normalized * speed;

}


void FixedUpdate()

{

rb.MovePosition(rb.position + moveVelocity * Time.fixedDeltaTime);

}

void OnCollisionEnter2D(Collision2D collision)

{

if (collision.gameObject.CompareTag("Wall"))

{

Debug.Log("Collided with wall");

}

}

}

You can also use layer masks to filter the results of a raycast.

You can create a layer mask using the following syntax:

int layerMask = 1 << LayerMask.NameToLayer("LayerName");

RaycastHit hit; if (Physics.Raycast(ray, out hit, Mathf.Infinity, layerMask)) { // code to execute if the ray hits a GameObject in the specified layer }

Seperation of concerns

Separation of concerns is a software design principle that promotes the separation of different aspects of a program into distinct sections.

In Unity, this can be achieved by organizing your code into different scripts and components.

You can create separate scripts for different aspects of your game, such as player movement, enemy AI, and UI management.

This allows you to keep your code organized and maintainable.

You can also use ScriptableObjects to store data and logic that can be shared between different scripts and components.

ScriptableObjects are a powerful feature in Unity that allows you to create reusable data containers that can be used in multiple places in your game.

MVC or MVVM patterns

MVC (Model-View-Controller) and MVVM (Model-View-ViewModel) are software design patterns that promote the separation of concerns in your code.

In Unity, you can use these patterns to organize your code and keep it maintainable.

In the MVC pattern, the Model represents the data and logic of your game, the View represents the user interface, and the Controller handles user input and updates the Model and View.

You can create a Model using a class that contains data and logic, a View using a UI component, and a Controller using a script that handles user input.

In the MVVM pattern, the Model represents the data and logic of your game, the View represents the user interface, and the ViewModel acts as an intermediary between the Model and View.

You can create a ViewModel using a class that contains data and logic, a View using a UI component, and a Model using a class that contains data and logic.

This allows you to keep your code organized and maintainable, while also promoting the separation of concerns in your code.

Scene loading (additive and single)

Scene loading is a powerful feature in Unity that allows you to load and unload scenes dynamically during runtime.

You can load scenes additively or as single scenes, depending on your needs.

To load a scene additively, you can use the following syntax:

SceneManager.LoadScene("SceneName", LoadSceneMode.Additive);

This loads the specified scene in addition to the currently loaded scenes.

You can also load a scene as a single scene using the following syntax:

SceneManager.LoadScene("SceneName", LoadSceneMode.Single);

This replaces the currently loaded scenes with the specified scene.

You can also unload a scene using the following syntax:

SceneManager.UnloadSceneAsync("SceneName");

This unloads the specified scene asynchronously, allowing you to continue executing code while the scene is being unloaded.

You can also use the SceneManager to manage multiple scenes in your game, allowing you to create complex game worlds with multiple levels and areas.
